**Convert modal to full-screen sheet (fixes FIX-01):**

The current form is a centered floating modal (`flex items-center justify-center p-4`) with `max-w-md`. On small screens, the emoji grid + form fields exceed viewport height, causing horizontal scroll when content overflows.

Redesign as a full-screen sheet:
1. Change the outer container from `flex items-center justify-center p-4` to a full-screen overlay:
   ```
   className="fixed inset-0 z-50 bg-game-bg/95 backdrop-blur-sm overflow-y-auto"
   ```
2. The inner form container: remove `max-w-md`, make it fill the screen:
   ```
   className="min-h-full w-full max-w-[480px] mx-auto bg-game-surface p-6"
   ```
   This gives the sheet feel — full height, content scrolls within it, matches the 480px app container width.
3. Remove the corner pixel decorations from the modal (they are a Phase 4 concern and add visual noise on a full-screen sheet)
4. Remove `imageRendering: 'pixelated'` and the floating `boxShadow` inline styles (not needed for full-screen sheet)

**Apply Steam-dark tokens:**

5. Replace all `bg-gray-900` / `bg-gray-800` with `bg-game-surface` / `bg-game-card`
6. Replace `border-gray-600` / `border-gray-700` with `border-game-border`
7. Replace all `style={{ fontFamily: 'monospace' }}` with `font-mono` Tailwind class
8. Replace `text-white` with `text-game-text`
9. Replace `text-gray-400` labels with `text-game-text-secondary`
10. Replace `text-gray-500` placeholders with `placeholder-game-text-secondary/50` (or `placeholder:text-game-text-secondary/50`)
11. Replace `focus:border-blue-500` with `focus:border-primary`

**Fix rarity selection buttons to use RARITY_COLORS tokens:**

12. The rarity selection grid already uses `RARITY_COLORS[r].border` and `RARITY_COLORS[r].bg` — these now resolve to `border-rarity-*` and `bg-rarity-*/15` from the updated tokens. This should work with no code change. Verify.
